# 电梯调度算法模拟器

> [English Documentation](./README.md)

完全自包含的浏览器端交互式电梯调度算法可视化工具。

---

## 功能特性

1. **双语界面 (EN/中)** — 点击右上角语言切换按钮，所有文字即时切换中/英文。
2. **明/暗主题** — 默认暗色，点击 🌙/☀️ 按钮切换。
3. **灵活的楼层与电梯数量** — 支持 2–60 层、1–6 台电梯，点击「应用」即时重建。
4. **四种调度算法** — FCFS、SCAN、LOOK、SSTF，可实时切换。
5. **服务模式** — 普通 / 分区 / 奇偶层，分区和奇偶层模式会限制电梯服务范围。
6. **可调分区分界层** — 分区模式下可拖动滑块调整低区与高区的分界楼层。
7. **乘客比例开门时长** — 进出乘客越多，开门停靠时间越长（最长 60 Ticks）。
8. **扩展统计** — 最大等待时间、空载率%、满载率%、电耗(kWh)、总楼层移动。
9. **A/B 算法对比** — 开启后并行运行两个算法，接收相同的乘客请求，实时比较性能指标。
10. **满员涌入预设** — 一键向随机楼层注入 25 名乘客，测试峰值压力。
11. **详细算法描述** — 每种算法的核心思想、工作原理、优缺点、适用场景（中英双语）。
12. **全面 SEO** — Open Graph、Twitter Card、JSON-LD 结构化数据。

## 调度算法说明

### FCFS (先来先服务)
按请求到达顺序依次服务。实现最简单，但高负载下后到乘客可能等待较长。

### SCAN (扫描算法 / 电梯算法)
向一个方向运行，服务沿途所有请求，到达端点后反向。对均匀分布的请求公平高效。

### LOOK
类似 SCAN，但只在最后一个待服务请求处反向，不再前往物理端点。比 SCAN 更高效。

### SSTF (最短寻道时间优先)
每次服务距当前位置最近的请求。吞吐量高，但负载不均衡时可能导致远端请求「饥饿」。

## 控件说明

| 控件 | 功能 |
|------|------|
| 开始 | 启动/继续模拟 |
| 暂停 | 暂停模拟 |
| 重置 | 清空并重建模拟 |
| 速度滑块 | 调整模拟速度（慢→快）|
| 自动 | 开启/关闭自动随机乘客生成 |
| A/B对比 | 开启/关闭并行算法对比 |
| 楼层输入 | 设置楼层数（2–60）|
| 电梯输入 | 设置电梯数（1–6）|
| 应用按钮 | 应用楼层/电梯数量变更 |
| ▲▼ 呼叫按钮 | 在指定楼层手动呼叫电梯 |
| 分区分界滑块 | 调整低区与高区分界楼层（仅分区模式）|
| 🌙/☀️ | 切换暗/亮主题 |
| EN/中 | 切换语言 |
